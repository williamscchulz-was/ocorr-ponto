Formulários & busca — toque, zoom e limpar

No celular, pequenos atritos se somam: limpar a busca apagando letra por letra, a tela dando zoom sozinha ao focar um campo, o teclado cobrindo o botão Salvar e filtros que se espremem em larguras irregulares. Cada ajuste aqui devolve um toque, um segundo ou um campo que ficou escondido — sem mudar nada do que o app faz.

01

Busca com botão limpar

O campo de busca já tem lupa à esquerda. Falta um caminho rápido para zerar a consulta — hoje só apagando manualmente. O botão limpar aparece apenas quando há texto.

Hoje
Sem botão limpar: o usuário segura o backspace ou seleciona tudo para recomeçar a busca.
Proposto
Toque o "x": ele limpa, esconde-se e devolve o foco ao input (teclado segue aberto). Touch target de 44px, ícone de 16px.
.toolbar__search .clear — absoluto à direita, width:44px (alvo de toque), visível só com .has-value; ao limpar, input.focus().
02

Contexto de resultado & “Mostrar mais”

Em listas grandes (130+ funcionários), renderizamos só ~50 itens e mostramos quantos estão visíveis. O resto entra por “Mostrar mais”, em append — a busca não re-dispara a animação de entrada (stagger), que roda só no 1º render.

Funcionários FioPulse
Mostrando 12 de 134

    Toque “Mostrar mais” — itens entram em append, sem re-animar a lista.

    Decisões

    • Render inicial limitado a ~50 itens — a rolagem inicial fica leve.
    • “Mostrando X de Y” dá contexto antes de o usuário rolar até o fim.
    • “Mostrar mais” faz append — não recria a lista do zero.
    • O stagger de entrada roda só no 1º render; digitar na busca não re-anima.

    Antes: a lista re-renderizava todos os 134 itens a cada tecla e sem paginação — travadas e re-stagger a cada letra.

    03

    Zoom-on-focus do iOS: 16px mata o zoom

    O Safari no iOS dá zoom automático ao focar qualquer input com fonte abaixo de 16px, desalinhando o layout. Subir todos os campos para 16px resolve sem JS e sem maximum-scale (que prejudica acessibilidade).

    Input a 15px 15px
    Dispara zoom no Safari iOS ao focar.
    Input a 16px 16px
    Sem zoom — a viewport fica estável.

    Alvos para 16px

    • .field input / select / textarea
    • .toolbar__search input
    • .toolbar select
    • .chat__composer textarea

    Se 15/14px for desejável no desktop por densidade, condicione com @media (pointer:fine){ ... font-size:15px } — mantendo 16px no toque.

    Bottom-sheet acima do teclado

    A folha de “Nova ocorrência” dimensionada em vh ignora o teclado virtual: ele cobre o último campo e o botão Salvar. Em dvh a folha encolhe acima do teclado e scrollIntoView centraliza o campo focado.

    .sheet { max-height: 92vh } — o footer cai sob o teclado.

    05

    Toolbar de filtros em 2 colunas

    No celular a toolbar em flex-wrap deixa a busca e dois selects competindo por largura — os selects encolhem para menos de 130px e o texto corta. Abaixo de 540px, a busca ocupa a 1ª linha inteira e os selects vão para um grid de 2 colunas.

    Funcionários FioPulse

    Selects abaixo de 130px — rótulos cortados (“Status: Ati…”, “Turno: 1º…”).

    Hoje — flex-wrap e larguras irregulares.

    Funcionários FioPulse

    Busca em largura total na 1ª linha; selects 1fr 1fr na 2ª, rótulos inteiros.

    Proposto — @media ≤540px: 2 linhas limpas.

    busca width:100% · selects grid-template-columns:1fr 1fr; gap:8px · padding 12px · font-size:16px.
